# Architecture Notes

## secp256k1 WASM multiplication

The secp256k1 WASM path is intentionally curve-specific. Generic Weierstrass code exposes an optional multiplication hook, and `src/secp256k1.ts` is the only curve that installs it.

The WASM module is generated by `scripts/wasm/secp256k1.mjs` with `@awasm/compiler`. The checked-in artifacts are:

- `src/wasm/secp256k1.wasm`: raw WebAssembly bytes for audit/reproducibility.
- `src/wasm/secp256k1.wasm.ts`: generated byte array plus AWASM memory offsets used by the TypeScript wrapper.
- `src/wasm/secp256k1.ts`: small synchronous loader and bigint marshaling wrapper.

The hot loop uses fixed-size integer limbs in WASM memory, not JavaScript `bigint`. Field elements are represented internally as sixteen 16-bit limbs, reduced modulo `p = 2^256 - 2^32 - 977`. The TypeScript wrapper performs the secp256k1 GLV scalar split, applies the beta endomorphism to the input projective point, and passes two signed half-size scalar multiplications into WASM.

The WASM function uses a fixed 4-bit window over projective complete addition/doubling formulas for `a = 0, b = 7`. Loop count, table scans, doublings, and additions do not depend on scalar bits; scalar windows only drive constant-time `select` operations. The helper can therefore replace both `multiply()` and `multiplyUnsafe()` while preserving constant-time behavior for secret scalars. If WebAssembly is unavailable, the hook returns `undefined` and the existing bigint implementation remains the fallback.

Current local microbenchmarks show this first WASM path is roughly neutral for `BASE.multiply` and slightly slower for arbitrary-point `multiplyUnsafe` against the bigint/endomorphism implementation, not yet the desired 2x speedup. Further speed work should focus on adding reusable precomputed base-point tables, reducing remaining point-operation call overhead, or moving to a wider limb backend with explicit carry handling.

Run `npm run build:wasm` after changing the AWASM source. The generator is deterministic; changes in the generated `.wasm` or `.wasm.ts` should be reviewed together with the source generator diff.

Use `npm run bench:ct:secp256k1` to compare coarse scalar-dependent timing classes for `secp256k1.Point.BASE.multiply`. This utility is a regression detector, not a proof of constant-time execution.
